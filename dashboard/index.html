Testing an Agent With No API (Emergent, Replit Agent, etc.) click to expand

Products like Emergent or Replit Agent don't expose a chat-completions API, so they can't be driven automatically like the providers above. src/manual_run.py handles this with a copy/paste workflow instead of skipping these products entirely:

  1. Export a fill-in-the-blanks prompt pack:
    python src/manual_run.py export --provider emergent --model "Emergent Agent" --out results/manual_emergent_pack.txt
  2. Open that file. For each block, paste the PROMPT into a fresh conversation with the agent, then paste its full reply in place of the RESPONSE placeholder line. Save when done.
  3. Import the filled-in pack back into the standard record format:
    python src/manual_run.py import --input results/manual_emergent_pack.txt --provider emergent --model "Emergent Agent" --out results/raw_responses_emergent.json
  4. Judge it exactly like any other run:
    python src/llm_judge.py --input results/raw_responses_emergent.json --output results/judged_emergent.json
    then pick it from "Previous runs" below to view it on this dashboard.

Import re-reads the whole file each time, so it's fine to do this in batches — anything still left as the placeholder is skipped with a warning rather than treated as a real response.
